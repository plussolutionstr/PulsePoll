Spike Koruması
Nginx rate limiting — IP başına kayıt 5 req/dk ile sınırlandırılır
Kayıt formu kuyruğa gider, DB'ye anlık 10k insert gelmez
Redis ile login token cache — DB'ye her istekte gitme
Anket listesi 5 dakika Redis'te kalır, her user için ayrı DB sorgusu yok
Kuyruk Stratejisi
Anlık cevap gerekmeyen her şey kuyruğa — kullanıcı beklemez
Push notification binlerce kişiye aynı anda gitmez, worker pace'li gönderir
SSI webhook geldiğinde kazanç işlemi asenkron — cüzdana yazar, bildirim atar
Worker crash olursa mesaj kaybolmaz, RabbitMQ'da bekler
Ölçeklenme Noktaları
Yük arttığında sadece API container'ı çoğaltılır — Nginx load balance eder
Worker container'ları bağımsız ölçeklenir
PostgreSQL için connection pooling (PgBouncer) eklenebilir
Tüm state dışarıda (Redis/DB) — API stateless, scale sorunsuz
